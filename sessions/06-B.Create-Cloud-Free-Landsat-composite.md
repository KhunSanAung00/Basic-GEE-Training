# Create Cloud-Free Landsat Composite

In this session, we will create an annual cloud-free Landsat composite in Google Earth Engine.

The workflow is:

1. Define an area of interest.
2. Select a year.
3. Load Landsat imagery.
4. Remove clouds, cloud shadows, snow, and fill pixels.
5. Build an annual median composite.
6. Display the final cloud-free image.

This tutorial uses **Landsat Collection 2 Surface Reflectance** because it is already atmospherically corrected and is recommended for land surface analysis.

## 1. Define the Area of Interest

First, create a point location and buffer it to make an area of interest. You can replace the coordinates with your own study area.

```javascript
// Example location: Bangkok, Thailand
var point = ee.Geometry.Point([100.5018, 13.7563]);

// Create a 20 km area around the point
var aoi = point.buffer(20000);

Map.centerObject(aoi, 10);
Map.addLayer(aoi, {color: 'red'}, 'Area of Interest');
```

## 2. Set the Year

We will create one annual composite. Change the `YEAR` value to create a composite for another year.

```javascript
var YEAR = 2023;

var startDate = ee.Date.fromYMD(YEAR, 1, 1);
var endDate = ee.Date.fromYMD(YEAR + 1, 1, 1);
```

## 3. Understand the Landsat QA Band

Landsat Collection 2 Surface Reflectance images include a quality band named `QA_PIXEL`.

This band stores cloud information using bit values. We can use these bit values to remove unwanted pixels:

- Bit 0 = fill
- Bit 1 = dilated cloud
- Bit 2 = cirrus
- Bit 3 = cloud
- Bit 4 = cloud shadow
- Bit 5 = snow

The cloud mask function below keeps only pixels where these bits are equal to 0.

## 4. Create a Reusable Cloud Mask Function

This function works for Landsat 4, 5, 7, 8, and 9 Collection 2 Surface Reflectance images.

```javascript
function maskLandsatSR(image) {
  var qa = image.select('QA_PIXEL');

  var fill = 1 << 0;
  var dilatedCloud = 1 << 1;
  var cirrus = 1 << 2;
  var cloud = 1 << 3;
  var cloudShadow = 1 << 4;
  var snow = 1 << 5;

  var mask = qa.bitwiseAnd(fill).eq(0)
      .and(qa.bitwiseAnd(dilatedCloud).eq(0))
      .and(qa.bitwiseAnd(cirrus).eq(0))
      .and(qa.bitwiseAnd(cloud).eq(0))
      .and(qa.bitwiseAnd(cloudShadow).eq(0))
      .and(qa.bitwiseAnd(snow).eq(0));

  return image.updateMask(mask);
}
```

## 5. Create a Reusable Scale Function

Surface Reflectance images need to be scaled before visualization or analysis. The reflectance bands start with `SR_B`.

```javascript
function scaleLandsatSR(image) {
  var opticalBands = image.select('SR_B.*')
      .multiply(0.0000275)
      .add(-0.2);

  return image.addBands(opticalBands, null, true);
}
```

## 6. Rename Bands for Different Landsat Sensors

Landsat 4, 5, and 7 use different band numbers from Landsat 8 and 9. To combine all Landsat sensors into one collection, rename the bands to common names.

### Landsat 4, 5, and 7

```javascript
function renameLandsat457(image) {
  return image.select(
    ['SR_B1', 'SR_B2', 'SR_B3', 'SR_B4', 'SR_B5', 'SR_B7'],
    ['blue', 'green', 'red', 'nir', 'swir1', 'swir2']
  );
}
```

### Landsat 8 and 9

```javascript
function renameLandsat89(image) {
  return image.select(
    ['SR_B2', 'SR_B3', 'SR_B4', 'SR_B5', 'SR_B6', 'SR_B7'],
    ['blue', 'green', 'red', 'nir', 'swir1', 'swir2']
  );
}
```

## 7. Create a Polymorphic Function to Load Landsat Surface Reflectance

This function lets you choose which Landsat sensors to use.

For example:

```javascript
// Use only Landsat 8 and Landsat 9
var landsat = getLandsatSR(['l8', 'l9'], aoi, startDate, endDate);

// Use Landsat 4, 5, 7, 8, and 9
var landsatAll = getLandsatSR(['l4', 'l5', 'l7', 'l8', 'l9'], aoi, startDate, endDate);
```

The sensor names are text values. If you prefer to write `getLandsatSR([l8, l9], aoi, startDate, endDate)`, define them first:

```javascript
var l8 = 'l8';
var l9 = 'l9';

var landsat = getLandsatSR([l8, l9], aoi, startDate, endDate);
```

This is similar to polymorphism because the same function can return different Landsat collections depending on the sensor list that you pass to it.

The `CLOUD_COVER` filter removes very cloudy scenes before processing. The pixel-level `QA_PIXEL` mask still removes cloud pixels inside each image.

```javascript
function prepareLandsatSR(collection, aoi, startDate, endDate, renameFunction) {
  return collection
      .filterBounds(aoi)
      .filterDate(startDate, endDate)
      .filter(ee.Filter.lt('CLOUD_COVER', 80))
      .map(maskLandsatSR)
      .map(scaleLandsatSR)
      .map(renameFunction);
}

function getLandsatSR(sensors, aoi, startDate, endDate) {
  sensors = sensors || ['l4', 'l5', 'l7', 'l8', 'l9'];

  var collections = {
    l4: prepareLandsatSR(ee.ImageCollection('LANDSAT/LT04/C02/T1_L2'), aoi, startDate, endDate, renameLandsat457),
    l5: prepareLandsatSR(ee.ImageCollection('LANDSAT/LT05/C02/T1_L2'), aoi, startDate, endDate, renameLandsat457),
    l7: prepareLandsatSR(ee.ImageCollection('LANDSAT/LE07/C02/T1_L2'), aoi, startDate, endDate, renameLandsat457),
    l8: prepareLandsatSR(ee.ImageCollection('LANDSAT/LC08/C02/T1_L2'), aoi, startDate, endDate, renameLandsat89),
    l9: prepareLandsatSR(ee.ImageCollection('LANDSAT/LC09/C02/T1_L2'), aoi, startDate, endDate, renameLandsat89)
  };

  var merged = ee.ImageCollection([]);

  sensors.forEach(function(sensor) {
    if (collections[sensor]) {
      merged = merged.merge(collections[sensor]);
    } else {
      print('Unknown Landsat sensor:', sensor);
    }
  });

  return merged;
}
```

## 8. Filter the Collection

Now use the function to create one Landsat collection for the selected year and location.

```javascript
// Use only Landsat 8 and Landsat 9
var landsat = getLandsatSR(['l8', 'l9'], aoi, startDate, endDate);

print('Cloud-masked Landsat collection:', landsat);
print('Number of images:', landsat.size());
```

## 9. Create an Annual Cloud-Free Composite

The `median()` reducer creates one image from all valid pixels in the year. This helps remove remaining clouds and noise.

```javascript
var annualComposite = landsat.median().clip(aoi);
```

## 10. Visualize the Composite

For true color visualization, use red, green, and blue bands.

```javascript
var trueColor = {
  bands: ['red', 'green', 'blue'],
  min: 0,
  max: 0.3
};

Map.addLayer(annualComposite, trueColor, 'Annual Landsat cloud-free composite');
```

You can also view a false color composite. This is useful for vegetation because healthy vegetation appears bright red.

```javascript
var falseColor = {
  bands: ['nir', 'red', 'green'],
  min: 0,
  max: 0.3
};

Map.addLayer(annualComposite, falseColor, 'False color composite');
```

## 12. Optional: Landsat TOA Example

Top of Atmosphere collections are useful for quick visualization, but Surface Reflectance is usually better for land analysis.

For Landsat TOA, cloud removal can use the `simpleCloudScore()` algorithm. This method is simple and useful for demonstration, but it is less precise than using the Surface Reflectance `QA_PIXEL` band.

```javascript
function maskLandsatTOA(image) {
  var cloudScore = ee.Algorithms.Landsat.simpleCloudScore(image);
  var cloud = cloudScore.select('cloud');
  var mask = cloud.lt(20);

  return image.updateMask(mask);
}

var landsat8Toa = ee.ImageCollection('LANDSAT/LC08/C02/T1_TOA')
    .filterBounds(aoi)
    .filterDate(startDate, endDate)
    .filter(ee.Filter.lt('CLOUD_COVER', 50))
    .map(maskLandsatTOA);

var toaComposite = landsat8Toa.median().clip(aoi);

Map.addLayer(
  toaComposite,
  {bands: ['B4', 'B3', 'B2'], min: 0.03, max: 0.25},
  'Landsat 8 TOA cloud-free composite'
);
```

## How To Modify This Script

- To change location, edit the coordinates in `ee.Geometry.Point([longitude, latitude])`.
- To change the analysis year, edit `YEAR`.
- To use a smaller or larger area, change `point.buffer(20000)`.
- To create a seasonal composite, replace the start and end dates with specific months.
- To choose sensors, edit the first argument of `getLandsatSR()`.
- Use `getLandsatSR(['l8', 'l9'], aoi, startDate, endDate)` for Landsat 8 and 9 only.
- Use `getLandsatSR(['l4', 'l5', 'l7', 'l8', 'l9'], aoi, startDate, endDate)` for all supported sensors.
- To create an index such as NDVI, use the common band names:

```javascript
var ndvi = annualComposite.normalizedDifference(['nir', 'red']).rename('NDVI');
Map.addLayer(ndvi, {min: 0, max: 1, palette: ['white', 'green']}, 'NDVI');
```

## Summary

In this tutorial, you created reusable functions for Landsat cloud removal and annual compositing:

- `maskLandsatSR()` removes cloud, shadow, snow, and fill pixels.
- `scaleLandsatSR()` applies Surface Reflectance scale factors.
- `renameLandsat457()` and `renameLandsat89()` standardize band names.
- `prepareLandsatSR()` applies the common filters and preprocessing steps to one collection.
- `getLandsatSR()` accepts a sensor list and returns one clean merged collection.
- `median()` creates the annual cloud-free composite.
